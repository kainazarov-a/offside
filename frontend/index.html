How it works

Three things, one screen

01 · WATCH

The sharp feed

TxLINE streams demargined match odds and score events for every World Cup fixture, seconds ahead of public sources. Each update is anchored on Solana by the provider.

02 · MEASURE

The crowd's delay

Polymarket prices arrive over a live socket with millisecond timestamps. When the sharp line jumps, OFFSIDE clocks how long the crowd takes to follow and logs every gap.

03 · PROVE

Every goal, on chain

Pick any goal and OFFSIDE fetches its Merkle proof, then asks the Txoracle program on Solana mainnet to verify it. The verdict comes from the chain, not from us.

Overview

OFFSIDE is a single process terminal that ingests two live sources at once: the TxLINE professional feed (odds plus match events, anchored on Solana) and Polymarket's public order book. It detects sharp information events with deterministic rules, measures how long the crowd takes to react, paper trades each dislocation with an honest execution model, and can prove any goal against the Txoracle program on Solana mainnet.

Everything on this site is real data from those feeds. There are no mocked numbers. The one exception is the labeled example walkthrough on the Home page.

Quickstart

npm install node tools/get_token.mjs --network mainnet --level 12 # one time, needs ~0.02 SOL for gas pip install httpx fastapi uvicorn websockets python backend/server.py --live # or --sim for a synthetic demo # open http://127.0.0.1:8000

The token script creates a throwaway local wallet, runs the on chain subscribe instruction for the free tier and stores API tokens under .secrets/. Nothing else touches the wallet.

  • Agent. Every signal the detector fired and every position currently open.

  • Replay. Rerun any finished match through the same pipeline at 30x.
  • Verify. Prove a goal against Solana. Details below.

Signals

A signal fires on a sharp implied probability jump above a configured threshold, or on a goal event. Rules are deterministic and printed in the config. There is no model guessing in the loop, which keeps every result reproducible in Replay.

Lag measurement

At signal time OFFSIDE snapshots the crowd price and the gap in probability points. It then watches the crowd feed for up to 60 seconds and records the time until the price converges toward the new sharp level. That number, in milliseconds, is one dislocation record. The histogram and the p50 and p90 stats are built from these records. Crowd prices arrive over Polymarket's websocket market channel, so timestamps are millisecond grade, with a REST poller as a fallback and a silence watchdog on top.

Paper trading model

When a signal fires with a big enough gap, the agent opens a paper position at the crowd price, but only after a configured execution latency, and it pays the spread both ways. The stake is fixed. Positions close on convergence or on a stop. The exact latency, spread and stake used are printed on the PnL tab next to the results, so every dollar shown is reproducible arithmetic, not a promise.

Goal detection

TxLINE encodes total goals per side in score message stats keys 1 and 2. OFFSIDE treats an increment as a goal. If a score drops because of a VAR decision the state rolls back without emitting a phantom goal. Penalty shootout goals live in separate keys and never touch the match score. Every detected goal is logged with its message sequence number, which is exactly what on chain verification needs.

On chain verification

TxLINE publishes daily Merkle roots of its score feed to Solana. Verification of one goal works like this:

  • Fetch the proof package from /api/scores/stat-validation?fixtureId&seq&statKeys.
  • Derive the account that stores that day's root: seeds ["daily_scores_roots", u16le(epochDay)] of program 9ExbZjAapQww1vfcisDmrngPinHTEfpjYRWMunJgcKaA.
  • Call validateStatV2 as a view. The program rebuilds the Merkle path from the stat leaf up to the stored root inside the Solana runtime and compares hashes.
  • Show the program's verdict, the root, the proof sizes and a Solscan link to the account.

The green PROVED badge is the program's answer, not ours. If the day's root has not been published yet, Verify says so and still shows the proof package.

API surface

OFFSIDE's own endpoints: /api/state, /api/stream (SSE), /api/replay/list, /api/replay/start, /api/replay/stop, /api/replay/status, /api/verify/candidates, /api/verify/goals, /api/verify/prove.

TxLINE endpoints used: guest auth and token activation, fixtures snapshot, odds snapshot, odds stream, scores stream, historical scores, hourly odds and fixtures updates, and stat validation. On chain: the subscribe instruction and the validateStatV2 view. Polymarket: the Gamma events API, the CLOB midpoint endpoint and the CLOB websocket market channel.

FAQ

Is this free to run?

Yes. The TxLINE free tier costs only a one time ~0.02 SOL gas fee for the on chain subscription. Polymarket data is public. The app is one Python process.

Does it place real bets?

No. Trading is paper only by design, and the execution model is printed next to every result.

Where do crowd prices come from?

Polymarket's public CLOB. Midpoint of best bid and ask, streamed over their websocket, polled over REST as a fallback.

Why doesn't Replay show trades?

The crowd feed has no public history at tick resolution, so a replay has nothing honest to trade against. Replay proves the ingestion and detection pipeline on real historical data and leaves the trader idle.

What if verification says the root is not on chain yet?

Roots are published per epoch day. For a very fresh match, pick a slightly older goal or check back later. The proof package itself is still shown.
